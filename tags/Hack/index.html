Luogu P1248 加工生产调度

题面：

题目描述

某工厂收到了n个产品的订单，这n个产品分别在A、B两个车间加工，并且必须先在A车间加工后才可以到B车间加工。

某个产品i在A、B两车间加工的时间分别为Ai、Bi。怎样安排这n个产品的加工顺序，才能使总的加工时间最短。这里所说的加工时间是指：从开始加工第一个产品到最后所有的产品都已在A、B两车间加工完毕的时间。

输入格式：

第一行仅—个数据n，表示产品的数量。

接下来n个数据是表示这n个产品在A车间加工各自所要的时间(都是整数)。

最后的n个数据是表示这n个产品在B车间加工各自所要的时间(都是整数)。

输出格式：

第一行一个数据，表示最少的加工时间；

第二行是一种最小加工时间的加工顺序。

数据范围：

$0\le n\le 1000$

思路:

显然,本题的产品选择是没有后效性的,可以直接贪心.

我们根据贪心的一贯做法,假设现在只有两个物品x,y,使其符合要求.设加工时间分别为$x_a,x_b,y_a,y_b$,先加工x,后加工y.

显然,加工的总时间为:

$x_a + max(x_b, y_a) + y_b$

继续推导得:

$$
x_a + max(x_b, y_a) + y_b < x_b + max(x_a, y_b) + y_a \
max(x_b, y_a) - x_b - y_a < max(x_a, y_b) - x_a - y_b \
max(x_a, y_b) < max(x_b, y_a)
$$

Mr.Dky就将这个式子作为了排序依据,这样重载了运算符:

于是顺利AC.

但是别忘了本文标题的”Hack”.这个策略实际上是有问题的,只是数据过水才水了过去.

$max(x_a, y_b) < max(x_b, y_a)$这个不等式是正确的,但它需要序列中的每一对产品都要满足.

我们直接在sort时使用它排序可以使得相邻的产品都满足这个不等式,但不相邻的呢?

这里引入一个概念:严格弱序(strict weak ordering).它是一个二元关系,是在集合S上的严格的部分序(一个不对称的不可逆或等价的传递关系).

根据C++标准,排序的运算符应当满足严格弱序.如果运算符不满足严格弱序,最终的运算结果事实上是不正确的(尽管它有时输出的是正确的答案).

严格弱序的四个条件:

  • $\forall x \not < x$(自反性).
  • $\forall x,y$,若$x < y$,则$y \not < x$(不对称性).
  • $\forall x,y,z$,若$x < y$且$y < z$,则$x < z$(传递性).
  • $\forall x,y,z$,若$x \not < y,y \not < x, y \not < z, z \not < y$,则$x \not < z, z \not < x$. (不可比性的传递性).

显然,我们得到的不等式是不满足不可比性的传递性的,当数值取很小时就已经存在反例了.

若该不等式满足不可比性的传递性,则满足以下命题:

$$\forall \left{
\begin{aligned}
max(x_a, y_b) = max(x_b, y_a)\
max(y_a, z_b) = max(y_b, z_a)\
\end{aligned}
\right.
,有max(x_a, z_b) = max(x_b, z_a).
$$

事实上当数值很小时就可以构造出反例…

eg:
$$
x_a = 3, x_b = 5, y_a = 7, y_b = 7, z_a = 2, z_b = 1\
$$
$$
\left{
\begin{aligned}
max(3, 7) = max(5, 7)\
max(7, 1) = max(7, 2)\
\end{aligned}
\right.
,但max(3, 1) \not = max(5, 2).
$$

反例其实很好构造,只需要令$y_a = y_b$为很大的数就可以构造出很多,因此该不等式不满足严格弱序.

由于该不等式不满足严格弱序,所以用它作为std::sort的比较函数是错误的.原因是在排序结束后,可能会出现前面的元素”大于”后面的元素的情况,这里的”大于”指不满足该不等式.

通过本题我们可以认识到,我们在重载小于号时一定要遵守严格弱序,否则我们的算法就是不严谨的,错误的.
